Introduction To HTML

The web, invented in 1990 is composed largely of HTML documents. HTML stands for Hypertext Markup Language and is the language the browsers use to interpret web pages. HTML is used to create electronic documents (called pages) that are displayed on the World Wide Web. We use HTML tags to create a simple HTML document that a browser can display as a web page.

Basically HTML is made up of :-

  1. Text content
  2. Markup
  3. Refrences
  4. Links to other pages

HTML Markup

HTML Tags

Most HTML elements consist of an opening and a closing tag with some content between them.
Its like :-

< TAG NAME (Opening) > Contents < /TAG NAME (Closing) >

For example,

<b> text </b> - b is for bold and it makes the 'text' content bold.
<em> text </em> - em is for emphasis and it makes the 'text' content italic.

HTML Attributes

Some elements accept additional values called attributes in their opening tags. Attributes provide additional information to the browser.
Its like :-

< TAG NAME ATTR(attribute name) = "value" > Contents < /TAG NAME >

For example,

<a> tag - The anchor tag takes an attribute called href that defines a URL to link - Creates a hyperlink to the href attribute value url and names it link.

Adding Images

<img> tag -



Its like :-

<img src="url" alt="text">

Displays the image hosted at the src attribute value "url" (this can also be a path to a local file on your computer).
If the image cannot be displayed, then the alt attribute value "text" is shown.
For more details, we can learn from internet.

Whitespace

HTML takes all text as one line, pressing enter makes no difference. So to input text in next line we use two tags specificly :-

  1. <br> tag - br stands for break. The text entered after this tag comes in the next line. Moreover, it is a void tag i.e. having no closing.
  2. <p> tag - p stands for paragraph. The text in this tag comes in form of paragraph. And, it is not void tag i.e. we need to close this tag.

Inline vs Block Elements

In HTML, all the spaces between some text comes in just one single space, so we put a <br> tag there which is inline.
While <p> tag is block i.e. makes a invisible box around text which is having height and length.

Creating A Structured Document

HTML's Tree-like Structure

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...
To know more about tree structures you can read them from internet.

More About

HTML

HyperText Markup Language - the standard markup language used to create web pages. It is a language with syntax and rules. Browser turns tags in html into elements that form a tree.

CSS

Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

DOM

Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

Boxifying The Design

Web pages are made with rectangular boxes that may contain all other forms. It's a good idea to think of all elements as boxes, for the ability to structure elements visually on a grid. So, in html, everything is a block. Programmers use Div tags to make those blocks and classes are used to identify and label each div tag(block).

Adding CSS Style And HTML Structure

Understanding CSS

Cascading Style Sheets is code written to define the "style" of HTML elements in a webpage. CSS can have multiple CSS files referenced in an HTML document. It creates the look and feel of a page. Cascading means you keep going down until you find the rule that is most specific to the element. It is an adaptation of the programming principle of inheritance.

Search and Replace

CSS can also be used as search and replace tool : you can identify a class or a tag of the element you want to find, and then apply that rule to it or do what you want to do with it, or what property values to replace with different ones.

Reducing Complexity

By utilizing CSS you can reduce the complexity and repetition of having to edit individual sections of HTML in order to change how it looks in the browser. Instead you just apply the desired style with CSS and as a result, it can be easily changed whenever necessary without having to edit the HTML extensively or at all.

Moreover, you can use various CSS selectors to define styles, which in turn changes the styling of corresponding HTML elements. A great resource to learn new CSS properties is: Mozilla CSS reference page.

CSS Vocabulary

  1. Style : This word can refer to many things and so it can be confusing. It can refer to :-
    • The HTML element.
    • The HTML attribute.
    • The general look of a web page.
  2. Rule : a line of CSS code describing the value that a certain attribute should take.
  3. Property : The property you want to change.
  4. Value : The value that you want to assign to the attribute.
  5. Selector : The name that you use to in order to target the elements that are assigned to a class or id attribute in the HTML.
  6. Class : A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks.
  7. ID : ID's are unique identifiers that uniquely identifies an element in HTML.

Box Sizing And Positioning

Box Sizing

There are four main points about box sizing :-

  1. HTML elements are boxes and each box has 4 components : margin, border, padding, content.
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
  3. There are two techniques you can use to help deal with sizing issues.
    • Set sizes in terms of percentages rather than pixels.
    • Set the box-sizing attribute to border-box for every element.
  4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

  1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
  2. Adding the rule display : flex; to the appropriate CSS will override this behaviour and let divs appear next to each other.

Code, Test, Refine

Here are steps to CODE your webpage :-

  1. Look for natural boxes : Build structure of the webpage by visually boxifying the given design.
  2. Look for repeated styles and semantic elements : Write your HTML, and add relevant headings.
  3. Apply CSS styles : Consider box sizing and positioning. Go from BIG to SMALL.
  4. Polish finer details : Fix HTML and CSS as needed.

Validating your code

Once you have your code completed, it is important to validate it using the W3C validation tools for HTML and CSS so that you can find mistakes that might make the pages appear differently on different browsers and find out if you are conforming to the HTML5 standard or not in your code.
